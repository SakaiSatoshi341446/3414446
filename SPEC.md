# 販売データ分析ダッシュボード 仕様書

## 1. 概要

本システムは、**サンプル販売データ**をもとに売上状況を可視化する、最小構成の販売データ分析ダッシュボードです。  
バックエンドは **Python + FastAPI**、フロントエンドは **React + TypeScript** で構成します。

目的は以下のとおりです。

- 売上データを簡単に確認できるダッシュボードを提供する
- 日別・商品別・地域別などの基本的な観点で売上を把握できるようにする
- 最小構成で実装可能な範囲に限定し、学習・検証用途に適した構成とする

---

## 2. 想定利用者

- 売上データの可視化を試したい開発者
- FastAPI / React + TypeScript 構成でのダッシュボード実装を学びたい受講者
- サンプルデータを使って分析画面の基本構成を確認したい利用者

## 3. スコープ

### 3.1 対象

- サンプル販売データの取得
- KPIの表示
- 売上推移の表示
- 商品別売上の表示
- 地域別売上の表示
- 期間指定による絞り込み

### 3.2 対象外

- 認証・認可
- 実データベース接続
- CSVアップロード機能
- 高度な分析機能（予測、異常検知など）
- 複雑な権限管理
- 印刷、エクスポート機能

---

## 4. システム構成

### 4.1 全体構成

- **フロントエンド**: React + TypeScript
- **バックエンド**: FastAPI
- **データ**: リポジトリ内に保持するサンプルJSONまたはCSV

### 4.2 構成方針

- 最小構成のため、バックエンドはサンプルデータをメモリ上で読み込んで返却する
- データベースは使用しない
- フロントエンドは単一画面、もしくは1ページ構成を基本とする
- グラフライブラリは任意（例: Recharts）とする

---

## 5. サンプルデータ仕様

### 5.1 データ項目

販売データは以下の項目を持つ想定です。

| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| date | string | 必須 | 売上日（YYYY-MM-DD） |
| region | string | 必須 | 地域名 |
| product | string | 必須 | 商品名 |
| category | string | 任意 | 商品カテゴリ |
| quantity | number | 必須 | 販売数量 |
| unit_price | number | 必須 | 単価 |
| sales | number | 必須 | 売上金額（quantity × unit_price） |

### 5.2 サンプルデータ例

```json
[
  {
    "date": "2026-01-01",
    "region": "East",
    "product": "Product A",
    "category": "Category 1",
    "quantity": 10,
    "unit_price": 1200,
    "sales": 12000
  },
  {
    "date": "2026-01-01",
    "region": "West",
    "product": "Product B",
    "category": "Category 2",
    "quantity": 5,
    "unit_price": 2000,
    "sales": 10000
  }
]
```

## 6. 機能要件

### 6.1 ダッシュボード表示

トップ画面に以下の情報を表示する。

#### KPI

- 総売上
- 総販売数量
- 注文件数（レコード件数）
- 平均売上（1レコードあたり）

#### グラフ・集計

- 日別売上推移
- 商品別売上ランキング
- 地域別売上集計

### 6.2 フィルタ機能

利用者は以下の条件でデータを絞り込める。

- 開始日
- 終了日
- 地域
- 商品名

### 6.3 データ取得API

フロントエンドはFastAPIからデータを取得する。

想定API:

- `GET /api/health` : ヘルスチェック
- `GET /api/sales` : 販売データ一覧取得
- `GET /api/dashboard/summary` : KPIサマリー取得
- `GET /api/dashboard/trend` : 日別売上推移取得
- `GET /api/dashboard/by-product` : 商品別売上取得
- `GET /api/dashboard/by-region` : 地域別売上取得

各APIは必要に応じてクエリパラメータとして以下を受け付ける。

- `start_date`
- `end_date`
- `region`
- `product`

## 9. 非機能要件

### 9.1 パフォーマンス

- サンプルデータ数は少量（数十〜数百件）を想定する
- APIレスポンスはローカル実行時に体感上即時であること

### 9.2 保守性

- フロントエンドとバックエンドの責務を分離する
- 型定義を明確にする
- APIレスポンス形式を統一する

### 9.3 可読性

- 学習用途を想定し、シンプルなディレクトリ構成とする
- 実装は最小限かつ理解しやすいことを優先する

## 11. 開発方針

- まずは固定のサンプルデータを表示できる状態を目標とする

以下を満たした場合、本仕様の最小構成を満たすものとする。

1. FastAPIでヘルスチェックおよび販売データ取得APIが動作する

3. KPIが画面上に表示される
4. 日別・商品別・地域別の集計結果が表示される
5. 期間・地域・商品による絞り込みができる
